# Solidity Integration

This guide explains how to integrate the Weiroll Python SDK with Solidity contracts for on-chain execution.

## Overview

The Weiroll Python SDK is used to create plans that can be executed by the Weiroll VM (a Solidity contract). The workflow is:

1. Use the Python SDK to create a plan
2. Send the encoded plan to the Weiroll VM contract
3. The VM executes the plan on-chain

## Core Components

### Weiroll VM

The Weiroll VM is a Solidity contract that executes plans. Its main function is:

```solidity
function execute(bytes32[] calldata commands, bytes[] calldata state) external;
```

### Plan Execution

The plan generated by the Python SDK consists of:

- `commands`: Array of encoded commands
- `state`: Array of state values

These are passed to the VM's `execute` function to execute the plan on-chain.

## Integration Example

### 1. Create a Plan with Python

First, create a plan using the Python SDK:

```python
from weiroll import Contract, Planner
from ape import Contract as ApeContract

# Create contract wrappers
token = Contract(ApeContract("0x6B175474E89094C44Da98b954EedeAC495271d0F"))
vault = Contract(ApeContract("0xd8063123BBA3B480569244AE66BFE72B6c84b00d"))

# Create a planner
planner = Planner()

# User address
user = "0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045"

# Add operations
balance = planner.add(token.balanceOf(user))
planner.add(vault.deposit(balance, user))

# Generate the plan
plan = planner.plan()
```

### 2. Deploy the Weiroll VM Contract

The Weiroll VM contract is deployed on various networks. You can also deploy your own instance.

### 3. Execute the Plan On-Chain

Using Ape to execute the plan:

```python
from ape import accounts, Contract

# Connect to the Weiroll VM contract
vm = Contract("0x1234...") # Weiroll VM address

# Get the plan
plan = planner.plan()

# Execute the plan (optional: convert hex strings to bytes)
commands = [bytes.fromhex(cmd[2:]) for cmd in plan["commands"]]
state = [bytes.fromhex(s[2:]) if s.startswith("0x") else s.encode() for s in plan["state"]]

# Send the transaction
account = accounts.load("deployer")
tx = vm.execute(commands, state, sender=account)
receipt = tx.wait()
```

## Custom VM Integration

For advanced use cases, you can create a custom contract that uses the Weiroll VM:

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.11;

import "@weiroll/weiroll/contracts/VM.sol";

contract MyWeirollStrategy {
    VM public immutable vm;
    
    constructor(address _vm) {
        vm = VM(_vm);
    }
    
    function executeStrategy(bytes32[] calldata commands, bytes[] calldata state) external {
        // Pre-execution logic
        
        // Execute the Weiroll plan
        vm.execute(commands, state);
        
        // Post-execution logic
    }
}
```

## Call Types in Solidity

The Weiroll VM supports different call types that correspond to these Solidity operations:

- `CALL` - Standard `call()` - External calls to other contracts
- `STATICCALL` - `staticcall()` - Read-only calls that can't modify state
- `DELEGATECALL` - `delegatecall()` - Execution in the caller's context
- `VALUECALL` - `call{value: amount}()` - Calls that send ETH

## Command Format

Each command is a `bytes32` value that encodes:

- Function selector (4 bytes)
- Call type and flags (1 byte)
- Input arguments (6 bytes)
- Output state slot (1 byte)
- Target contract address (20 bytes)

The Weiroll VM decodes these commands and executes them in sequence.

## State Management

The state array holds all values used or produced by the commands:

- Input values (like addresses and amounts)
- Output values from function calls
- Values passed between commands

The VM manages this state array during execution.

## Gas Considerations

Weiroll plans are optimized for gas efficiency, but consider:

- Ordering operations to minimize state size
- Using appropriate call types (STATICCALL for reads)
- Deduplicating repeated values in state
- Testing gas usage before deployment

## Security Considerations

When integrating Weiroll with Solidity:

- Validate all inputs before execution
- Consider access control for execution functions
- Test extensively on testnets
- Audit contracts that use Weiroll VM
- Verify the commands and state match expected patterns
